
#### Integrations

##### Vega

- Updated alert and incident status fetch filters to show human-readable labels (e.g. **IN PROGRESS**) while sending Vega API enum values (e.g. `IN_PROGRESS`) in requests.
- Ignored invalid custom values entered in status multi-select fields so they are not sent to the API.
- Applied the same validation to alert and incident severity and verdict fetch filters so only supported Vega values are sent to the API.
- Improved test connection error messages to report Base URL connectivity issues separately from invalid credentials.
